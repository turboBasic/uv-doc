## Summary

`uv pip compile` resolves a set of dependency inputs and writes pinned versions to a
`requirements.txt` (or `pylock.toml`) output file ÔÇö the uv equivalent of `pip-compile`
from pip-tools.

## Syntax / Usage

```bash
uv pip compile [OPTIONS] <SRC_FILE|--group <GROUP>>
```

## Details

`uv pip compile` accepts one or more source files and resolves all transitive dependencies
against the configured index, then writes the result to stdout or to a file with `-o`.

**Supported input formats:**

- `requirements.in` or any `requirements.txt`-like file
- `pyproject.toml` ÔÇö extracts `[project].dependencies`; use `--extra` or `--all-extras`
  for optional dependencies
- `setup.py` / `setup.cfg` ÔÇö extracts dependencies from legacy project definitions
- `.py` files with PEP 723 inline metadata
- `-` ÔÇö reads requirements from stdin

Multiple source files can be provided; their order determines resolution priority.

**Output file (`-o` / `--output-file`):**

Without `-o`, output is written to stdout only. When an output file already exists, uv
reads the previously pinned versions and keeps them unless `--upgrade` or
`--upgrade-package` is also passed. The output format is inferred from the file extension
(`.txt` Ôćĺ `requirements.txt`, `.toml` Ôćĺ `pylock.toml`); override with `--format`.

**Extras and dependency groups:**

- `--extra <name>` ÔÇö include a named optional-dependency group from `pyproject.toml`;
  repeatable.
- `--all-extras` ÔÇö include all optional-dependency groups. Extras are not supported with
  plain `requirements.in` input.
- `--group <name>` ÔÇö include a PEP 735 dependency group from the `pyproject.toml` in the
  working directory (or from a path qualified as `some/path/pyproject.toml:groupname`).
  `--group` always sources from the local `pyproject.toml`, not from a path supplied as a
  positional `SRC_FILE`.

**Upgrade behavior:**

- `--upgrade` / `-U` ÔÇö ignore all pinned versions in the existing output file and re-resolve
  everything to the latest compatible versions. Implies `--refresh`.
- `--upgrade-package <pkg>` / `-P <pkg>` ÔÇö upgrade only the named package; repeatable.

**Constraints and overrides:**

- `--constraint <file>` / `-c` ÔÇö add version bounds without triggering installation of the
  constrained packages. Constraints are additive. `UV_CONSTRAINT` env var also accepted.
  uv reads `constraint-dependencies` from the workspace root `pyproject.toml` and appends
  them to any file-based constraints.
- `--build-constraint <file>` / `-b` ÔÇö same semantics but applies only to build-time
  dependencies. Use this instead of `--constraint` when you want to pin build tools such
  as `setuptools`. `UV_BUILD_CONSTRAINT` env var also accepted.
- `--override <file>` ÔÇö force a specific version regardless of what constituent packages
  declare. Overrides are absolute (they replace, not add to, package requirements).
  `UV_OVERRIDE` env var also accepted.

**Annotation style:**

Comments in the output file show which package required each dependency.
`--annotation-style` accepts `split` (default, one line per source) or `line`
(comma-separated on a single line). Pass `--no-annotate` to omit annotations entirely.
Pass `--no-header` to omit the autogenerated header comment.

**Hashes:**

`--generate-hashes` appends a `--hash=sha256:ÔÇŽ` entry for every resolved distribution,
enabling hash-checked installs downstream.

**Extras stripping:**

By default uv strips extras from output (equivalent to `--strip-extras`). Pass
`--no-strip-extras` to preserve them. Note: output files produced with `--no-strip-extras`
cannot be used as constraints files in subsequent `install` or `sync` calls.

**Index URL emission:**

By default uv does not write index URLs into the output file. Pass `--emit-index-url` to
include `--index-url` and `--extra-index-url` lines; unlike pip-compile, uv includes the
default PyPI URL as well when this flag is active.

**Custom header comment:**

`--custom-compile-command <text>` replaces the default `uv pip compile ÔÇŽ` header with
the provided string. Useful when `uv pip compile` is wrapped in a script and you want the
output file to advertise the wrapper command. Also settable via `UV_CUSTOM_COMPILE_COMMAND`.

**Universal resolution:**

`--universal` produces a single output file compatible with all OS/architecture/Python
combinations by retaining environment markers. Implies `--no-strip-markers`.

## Examples

```bash
# Lock pyproject.toml dependencies to a file
uv pip compile pyproject.toml -o requirements.txt

# Lock a requirements.in file
uv pip compile requirements.in -o requirements.txt

# Include optional extras
uv pip compile pyproject.toml --extra dev --extra docs -o requirements-dev.txt

# Include all extras
uv pip compile pyproject.toml --all-extras -o requirements-all.txt

# Lock a dependency group
uv pip compile --group dev -o requirements-dev.txt

# Read from stdin
echo "flask\nrequests" | uv pip compile - -o requirements.txt

# Upgrade everything, ignoring existing pins
uv pip compile requirements.in -o requirements.txt --upgrade

# Upgrade a single package
uv pip compile requirements.in -o requirements.txt --upgrade-package flask

# Apply version constraints without adding new packages
uv pip compile requirements.in --constraint constraints.txt -o requirements.txt

# Apply build-time constraints (e.g. pin setuptools for all builds)
uv pip compile requirements.in --build-constraint build-constraints.txt -o requirements.txt

# Override a transitive dependency version
uv pip compile requirements.in --override overrides.txt -o requirements.txt

# Generate hashes for verified installs
uv pip compile requirements.in --generate-hashes -o requirements.txt

# Preserve extras in output (note: result cannot be used as a constraints file)
uv pip compile pyproject.toml --no-strip-extras -o requirements.txt

# Embed index URLs in output and use a custom header
uv pip compile requirements.in --emit-index-url \
    --custom-compile-command "scripts/compile.sh" -o requirements.txt

# Universal resolution (cross-platform output with markers intact)
uv pip compile pyproject.toml --universal -o requirements.txt
```

## Caveats / Common Mistakes

- Without `-o`, output goes to stdout only. The file is not written automatically ÔÇö unlike
  `pip-compile`, which defaults to writing `requirements.txt`.
- uv defaults to `--strip-extras`; `pip-compile` defaults to `--no-strip-extras`. If you
  are migrating from pip-tools and need extras preserved, add `--no-strip-extras` explicitly.
- uv does not emit index URLs by default; `pip-compile` does (for non-PyPI indexes). Pass
  `--emit-index-url` to reproduce that behavior ÔÇö but note uv will then also emit the
  default PyPI URL.
- `--group` always reads the dependency group from the `pyproject.toml` in the current
  working directory, not from a positional `SRC_FILE` path.
- `--constraint` does not apply to build dependencies. Use `--build-constraint` to pin
  build tools like `setuptools` or `wheel`.
- Output files generated with `--no-strip-extras` cannot be used as input to
  `uv pip install -c` or `uv pip sync -c`.
- The ordering of input files (and requirements within them) influences which package
  versions are selected, because uv uses input order as a resolution priority signal.
